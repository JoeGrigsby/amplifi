## Brand Hierarchy

```
AmpliFi  (mission + infrastructure)
├── The Gruene Songwriters Collective  (soul + proof of concept)
├── The Songwriter Trail  (the road between artists and America)
├── The AmpliFi Studio Network
└── The AmpliFi Knowledge Hub
```

Each program has its own voice and visual identity. All report to AmpliFi's core purpose: **for the song.**

---

## How to Use These Documents

**Writing copy or content:** Read the Tone of Voice and Sounds Like / Doesn't Sound Like sections for the relevant brand before you write anything.

**Briefing a designer or agency:** Share the full document for the relevant brand. The Visual Identity section covers palette, typography, photography, and mood.

**Briefing a collaborator new to the project:** Start with the AmpliFi document for the overall mission, then the specific program document for context.

**Uploading to Claude for AI-assisted content:** Attach the relevant markdown file(s) at the start of any session where you're creating on-brand content.

---

## Updating These Documents

These are living documents. Update them when:
- A program evolves its positioning or adds new messaging
- New visual identity decisions are made
- The brand hierarchy changes (e.g. a new collective is added)
- Voice guidelines are refined based on what's working

When updating, note the date in the *Last updated* line at the bottom of each file.
